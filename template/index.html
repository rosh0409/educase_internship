API Documentation

This documentation provides detailed instructions for using the /addSchool and /listSchools endpoints locally.

1. Add School API

Endpoint:

POST /addSchool

Request Body:

The following fields are required in the JSON payload:


    {
        "name": "string (required)",
        "address": "string (required)",
        "latitude": "float (required)",
        "longitude": "float (required)"
    }

Example Request:


    POST http://localhost:8000/addSchool
    Content-Type: application/json

    {
        "name": "Greenwood High",
        "address": "123 Main Street, Springfield",
        "latitude": 37.7749,
        "longitude": -122.4194
    }

Example Response:


    {
        "message": "School added successfully",
        "id": 1
    }

2. List Schools API

Option 1: Using Query Parameters

Endpoint:

GET /listSchools

Query Parameters:

Include these query parameters in the URL:


    latitude: float (required)
    longitude: float (required)

Example Request:


    GET http://localhost:8000/listSchools?latitude=37.7749&longitude=-122.4194

Option 2: Using Route Parameters

Endpoint:

GET /listSchools/:latitude/:longitude

Example Request:


    GET http://localhost:8000/listSchools/37.7749/-122.4194

Example Response:


    [
        {
            "id": 1,
            "name": "Greenwood High",
            "address": "123 Main Street, Springfield",
            "latitude": 37.7749,
            "longitude": -122.4194,
            "distance": 0.0
        },
        {
            "id": 2,
            "name": "Blue Ridge School",
            "address": "456 Elm Street, Sunnyvale",
            "latitude": 37.7849,
            "longitude": -122.4094,
            "distance": 1.2
        }
    ]
